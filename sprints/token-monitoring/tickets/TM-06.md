# TM-06: Frontend — Review History Table Component

## Description
Build a `ReviewHistory` component that displays a paginated table of past reviews for an installation. Users can see which PRs were reviewed, token usage per review, status, and duration.

## Location
- **New**: `frontend/src/components/ReviewHistory.tsx`
- **Modify**: `frontend/src/routes/settings.$installationId.tsx` — add a "Review History" tab/section below the usage dashboard

## Design

### Table Columns
| Column | Source Field | Notes |
|--------|-------------|-------|
| Repository | `repoFullName` | e.g. `acme/widget` |
| PR | `prNumber` + `prTitle` | Link to GitHub PR if possible |
| Provider / Model | `llmProvider` + `llmModel` | Badge for provider |
| Tokens | `promptTokens` + `completionTokens` | Show as "3.2k / 1.1k" |
| Comments | `inlineCommentCount` | Number of inline comments posted |
| Status | `status` | Badge: green=completed, red=failed, yellow=pending |
| Duration | `durationMs` | Formatted as "2.8s" |
| Date | `createdAt` | Relative time (e.g. "2 hours ago") |

### Pagination
- "Previous" / "Next" buttons at the bottom
- Show "Page X of Y" text
- Default 20 rows per page

### Status Filter
- Dropdown/tabs: All, Completed, Failed
- Updates the `status` query param

## Data Fetching
- Fetch from `GET /api/installations/:id/reviews?page=1&limit=20&status=...`
- Show skeleton rows while loading

## Acceptance Criteria
- [ ] Table renders with all columns correctly populated
- [ ] Pagination works (previous/next, page indicator)
- [ ] Status filter updates the displayed results
- [ ] Null token values display as "—" (for failed reviews)
- [ ] Loading skeletons during fetch
- [ ] Empty state message when no reviews exist
