# TM-01: Backend — Token Usage Stats API Endpoint

## Description
Create an authenticated API endpoint that returns aggregated token usage statistics for a given installation. This is the primary data source for the frontend usage dashboard.

## Endpoint
```
GET /api/installations/:id/stats?from=<ISO date>&to=<ISO date>
```

## Response Shape
```json
{
  "totals": {
    "promptTokens": 125000,
    "completionTokens": 43000,
    "totalTokens": 168000,
    "reviewCount": 47,
    "failedCount": 2,
    "avgDurationMs": 3200
  },
  "byProvider": [
    {
      "provider": "openai",
      "model": "gpt-4o",
      "promptTokens": 80000,
      "completionTokens": 28000,
      "reviewCount": 30
    }
  ],
  "daily": [
    {
      "date": "2026-02-20",
      "promptTokens": 12000,
      "completionTokens": 4300,
      "reviewCount": 5
    }
  ]
}
```

## Affected Files
- **New**: `src/api/stats.ts` — new Elysia plugin with the stats route
- **Modify**: `src/index.ts` — register the new plugin
- **Read**: `src/db/schema.ts` — query the `reviews` table

## Implementation Notes
- Use Drizzle ORM aggregation functions: `sum()`, `count()`, `avg()`
- Filter by `reviews.installationId` and `createdAt` within the date range
- Group by `llmProvider` + `llmModel` for the `byProvider` breakdown
- Group by `DATE(created_at)` for the `daily` breakdown
- Only include reviews with `status = 'completed'` for token totals (failed reviews have null tokens)
- Require authenticated session — reuse the session check pattern from `src/api/installations.ts`
- Verify user has access to the installation via `userHasInstallationAccess`

## Acceptance Criteria
- [ ] Returns correct aggregated totals for an installation
- [ ] `byProvider` breakdown groups by provider + model
- [ ] `daily` array returns one entry per day in the date range
- [ ] Defaults to last 30 days if no date range provided
- [ ] Returns 401 if not authenticated
- [ ] Returns 403 if user lacks access to the installation
- [ ] Returns empty totals (zeros) if no reviews exist
