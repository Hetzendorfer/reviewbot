# TM-03: Backend — Review History Endpoint per Installation

## Description
Create an authenticated endpoint that returns a paginated list of past reviews for an installation. This powers the review history table in the frontend.

## Endpoint
```
GET /api/installations/:id/reviews?page=1&limit=20&status=completed
```

## Response Shape
```json
{
  "reviews": [
    {
      "id": 42,
      "repoFullName": "acme/widget",
      "prNumber": 123,
      "prTitle": "Add caching layer",
      "commitSha": "abc1234",
      "llmProvider": "openai",
      "llmModel": "gpt-4o",
      "promptTokens": 3200,
      "completionTokens": 1100,
      "inlineCommentCount": 4,
      "status": "completed",
      "durationMs": 2800,
      "errorMessage": null,
      "createdAt": "2026-02-20T14:30:00Z"
    }
  ],
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 47,
    "totalPages": 3
  }
}
```

## Affected Files
- **Modify**: `src/api/stats.ts` (created in TM-01) — add the reviews list route to the same plugin
- **Read**: `src/db/schema.ts` — query `reviews` table filtered by `installationId`

## Implementation Notes
- Use Drizzle `select().from(reviews).where(...).orderBy(desc(reviews.createdAt)).limit(limit).offset((page - 1) * limit)`
- Separate count query for pagination: `SELECT count(*) FROM reviews WHERE installation_id = ?`
- Optional `status` filter param (all, completed, failed, pending)
- Require authentication + installation access check (same pattern as TM-01)
- Cap `limit` at 100 to prevent abuse

## Acceptance Criteria
- [ ] Returns paginated list of reviews for the installation
- [ ] Supports filtering by status
- [ ] Pagination metadata is correct (total, totalPages)
- [ ] Default page=1, limit=20 when params omitted
- [ ] Ordered by `createdAt` descending (most recent first)
- [ ] Returns 401/403 for unauthorized access
