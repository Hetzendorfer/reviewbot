# TM-05: Frontend — Usage Dashboard Component

## Description
Build a `UsageDashboard` component that visualizes token consumption stats for an installation. This is the primary view for monitoring LLM costs and usage trends.

## Location
- **New**: `frontend/src/components/UsageDashboard.tsx`
- **Modify**: `frontend/src/routes/settings.$installationId.tsx` — add a "Usage" tab/section that renders `UsageDashboard`

## Design
The dashboard should include:

1. **Summary Cards** (top row)
   - Total tokens used (prompt + completion)
   - Total reviews count
   - Average review duration
   - Failed review count
   Use shadcn `Card` components already available in `frontend/src/components/ui/card.tsx`.

2. **Provider Breakdown** (middle section)
   - Table or card list showing per-provider/model token usage
   - Columns: Provider, Model, Reviews, Prompt Tokens, Completion Tokens
   - Use shadcn `Badge` for provider names

3. **Daily Usage Chart** (bottom section)
   - Simple bar or area chart showing daily token usage over the selected date range
   - Consider a lightweight chart library (e.g., `recharts`) or a simple CSS-based bar chart to avoid heavy dependencies

4. **Date Range Picker**
   - Default: last 30 days
   - Options: 7d, 30d, 90d, custom range

## Data Fetching
- Fetch from `GET /api/installations/:id/stats?from=...&to=...`
- Use TanStack Query or a simple `useEffect` + `fetch` pattern consistent with the existing codebase
- Show `Skeleton` components (already available) while loading

## Acceptance Criteria
- [ ] Summary cards show correct totals
- [ ] Provider breakdown table renders with correct data
- [ ] Daily usage visualization shows trends
- [ ] Date range selection updates the data
- [ ] Loading skeletons shown while fetching
- [ ] Error state shown if the API call fails
- [ ] Responsive layout (works on mobile)
