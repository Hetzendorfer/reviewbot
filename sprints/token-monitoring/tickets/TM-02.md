# TM-02: Backend тАФ Extend `/metrics` with Token Aggregations

## Description
The existing `/metrics` endpoint (`src/index.ts`) returns review counts and average duration but no token data. Add global token aggregations so operators can monitor overall LLM consumption, and protect the endpoint with a bearer token for internal use.

## Current Response
```json
{
  "reviews": { "total": 120, "failed": 3, "avgDurationMs": 2800 },
  "queue": { "pending": 1, "processing": 0, "failed": 0 }
}
```

## Target Response
```json
{
  "reviews": {
    "total": 120,
    "failed": 3,
    "avgDurationMs": 2800
  },
  "tokens": {
    "totalPromptTokens": 500000,
    "totalCompletionTokens": 175000,
    "totalTokens": 675000
  },
  "queue": { "pending": 1, "processing": 0, "failed": 0 }
}
```

## Affected Files
- **Modify**: `src/index.ts` тАФ extend the `/metrics` handler with a `SUM` query on `prompt_tokens` and `completion_tokens`
- **Modify**: `src/config.ts` тАФ add `METRICS_TOKEN` env var
- **Modify**: `.env.example` тАФ document `METRICS_TOKEN`

## Implementation Notes
- Single additional SQL query: `SELECT SUM(prompt_tokens), SUM(completion_tokens) FROM reviews WHERE status = 'completed'`
- Require `Authorization: Bearer <token>` for `/metrics`
- Configure token via `METRICS_TOKEN`
- Compare tokens with timing-safe comparison (`timingSafeEqual`)
- If `METRICS_TOKEN` is missing in production, disable `/metrics` (return `404`)
- Consider adding a `tokensLast24h` sub-object for recent usage trends

## Acceptance Criteria
- [ ] `/metrics` response includes a `tokens` object with `totalPromptTokens`, `totalCompletionTokens`, `totalTokens`
- [ ] Null token values (from failed reviews) are excluded from the sum
- [ ] No breaking changes to the existing response shape
- [ ] `/metrics` returns `401` when auth header is missing/invalid
- [ ] Metrics token validation uses timing-safe comparison
- [ ] `/metrics` is disabled in production when `METRICS_TOKEN` is not configured
