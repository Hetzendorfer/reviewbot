# TM-02: Backend тАФ Extend `/metrics` with Token Aggregations

## Description
The existing `/metrics` endpoint (`src/index.ts`) returns review counts and average duration but no token data. Add global token aggregations so operators can monitor overall LLM consumption.

## Current Response
```json
{
  "reviews": { "total": 120, "failed": 3, "avgDurationMs": 2800 },
  "queue": { "pending": 1, "processing": 0, "failed": 0 }
}
```

## Target Response
```json
{
  "reviews": {
    "total": 120,
    "failed": 3,
    "avgDurationMs": 2800
  },
  "tokens": {
    "totalPromptTokens": 500000,
    "totalCompletionTokens": 175000,
    "totalTokens": 675000
  },
  "queue": { "pending": 1, "processing": 0, "failed": 0 }
}
```

## Affected Files
- **Modify**: `src/index.ts` тАФ extend the `/metrics` handler with a `SUM` query on `prompt_tokens` and `completion_tokens`

## Implementation Notes
- Single additional SQL query: `SELECT SUM(prompt_tokens), SUM(completion_tokens) FROM reviews WHERE status = 'completed'`
- Keep this endpoint unauthenticated (it's an ops/monitoring endpoint, consistent with current behavior)
- Consider adding a `tokensLast24h` sub-object for recent usage trends

## Acceptance Criteria
- [ ] `/metrics` response includes a `tokens` object with `totalPromptTokens`, `totalCompletionTokens`, `totalTokens`
- [ ] Null token values (from failed reviews) are excluded from the sum
- [ ] No breaking changes to the existing response shape

## Depends On
None

## Effort
S (small, < 2h)
