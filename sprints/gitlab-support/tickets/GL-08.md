# GL-08: Frontend — Multi-Platform Support in UI

## Description
Update the frontend to support both GitHub and GitLab users. This includes a platform-aware login page, installation management for GitLab projects, and visual indicators for the platform.

## Affected Files
- **Modify**: `frontend/src/components/LoginView.tsx` — add GitLab login button
- **Modify**: `frontend/src/components/DashboardView.tsx` — show platform badge on installations
- **Modify**: `frontend/src/components/InstallationCard.tsx` — platform icon/badge
- **Modify**: `frontend/src/components/SettingsView.tsx` — GitLab-specific instructions if needed
- **Modify**: `frontend/src/types/index.ts` — add `platform` field to types
- **New**: `frontend/src/components/GitLabSetup.tsx` — guide for setting up GitLab webhook (since there's no "install app" flow like GitHub)

## Changes

### Login Page
- Add "Sign in with GitLab" button below the existing GitHub button
- GitLab button links to `/api/auth/gitlab`
- Use GitLab brand color (#FC6D26) for the button

### Dashboard
- `InstallationCard` shows a GitHub/GitLab icon or badge next to the account name
- `/api/auth/me` response includes `platform` so the frontend knows which login was used

### Installation Management
GitHub and GitLab have different setup flows:
- **GitHub**: "Install GitHub App" button → GitHub redirects back after installation
- **GitLab**: Manual setup — user enters their GitLab project/group URL, and ReviewBot provides a webhook URL + secret to configure in GitLab settings

Create a `GitLabSetup` component that:
1. Shows the webhook URL: `https://<base-url>/webhooks/gitlab`
2. Generates and displays a webhook secret
3. Provides step-by-step instructions for adding the webhook in GitLab (Settings → Webhooks)
4. Has a "Verify Connection" button that checks if the webhook is configured correctly

### Types
Add `platform: 'github' | 'gitlab'` to:
- `Installation` type
- `User` / auth response type

## Acceptance Criteria
- [ ] Login page shows both GitHub and GitLab login options
- [ ] Dashboard shows platform badge on each installation
- [ ] GitLab setup flow guides user through webhook configuration
- [ ] Settings page works for both GitHub and GitLab installations
- [ ] Platform field is available in all relevant frontend types
- [ ] UI is responsive and consistent with existing design

- GL-03 (DB must support platform field)
