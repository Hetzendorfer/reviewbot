# GL-03: DB Schema Migration (Multi-Platform Support)

## Description
Migrate the database schema to support multiple platforms. Currently, the schema has GitHub-specific column names (`github_installation_id`, `github_account_login`, etc.). These need to be generalized with a `platform` discriminator column.

## Affected Files
- **Modify**: `src/db/schema.ts` — rename columns, add `platform` enum
- **New**: Drizzle migration file (generated via `bun run db:generate`)
- **Modify**: All files querying the renamed columns (pipeline, installations API, auth)

## Schema Changes

### New enum
```sql
CREATE TYPE platform AS ENUM ('github', 'gitlab');
```

### `installations` table
| Current Column | New Column | Notes |
|---------------|-----------|-------|
| `github_installation_id` | `platform_installation_id` | `text` (GitLab project IDs are numeric but groups are different) |
| `github_account_login` | `account_login` | Drop `github_` prefix |
| `github_account_type` | `account_type` | Drop `github_` prefix |
| _(new)_ | `platform` | `platform` enum, NOT NULL, default `'github'` |

### `review_jobs` table
| Current Column | New Column | Notes |
|---------------|-----------|-------|
| `check_run_id` (bigint) | `platform_status_id` (text) | Text to accommodate different ID formats |
| _(new)_ | `platform` | `platform` enum, NOT NULL |

### `sessions` table
| Current Column | New Column | Notes |
|---------------|-----------|-------|
| `github_user_id` | `platform_user_id` | integer |
| `github_username` | `platform_username` | text |
| `github_avatar` | `platform_avatar` | text |
| _(new)_ | `platform` | `platform` enum, NOT NULL, default `'github'` |

## Migration Strategy
1. Add new columns with defaults
2. Copy data from old columns to new columns
3. Drop old columns
4. Add NOT NULL constraints on new columns

This is a **breaking migration** — run it during a maintenance window. Existing rows get `platform = 'github'` by default.

## Acceptance Criteria
- [ ] Migration runs cleanly on an existing database with data
- [ ] All existing queries updated to use new column names
- [ ] `platform` column defaults to `'github'` for backward compatibility
- [ ] `platform_installation_id` is unique per platform (composite unique constraint: `platform` + `platform_installation_id`)
- [ ] Drizzle schema matches the migrated DB structure
- [ ] `bunx tsc --noEmit` passes after all renames
