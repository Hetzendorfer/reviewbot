# GL-04: Implement GitLab OAuth Authentication

## Description
Implement GitLab OAuth 2.0 flow so users can log in with their GitLab account, similar to the existing GitHub OAuth flow in `src/api/auth.ts`.

## Affected Files
- **New**: `src/platforms/gitlab/auth.ts` — GitLab OAuth implementation
- **Modify**: `src/api/auth.ts` — add `/api/auth/gitlab` and `/api/auth/gitlab/callback` routes alongside existing GitHub routes
- **Modify**: `src/config.ts` — add GitLab OAuth env vars (see GL-09)

## GitLab OAuth Flow
1. **Redirect**: `GET /api/auth/gitlab` → redirect to `https://gitlab.com/oauth/authorize?client_id=...&redirect_uri=...&response_type=code&scope=read_user+api&state=...`
2. **Callback**: `GET /api/auth/gitlab/callback` → exchange code for token at `https://gitlab.com/oauth/token`
3. **User info**: `GET https://gitlab.com/api/v4/user` with `Authorization: Bearer <token>`
4. **Session**: Create session with `platform: 'gitlab'`, `platform_user_id`, `platform_username`, `platform_avatar`

## Implementation Notes
- GitLab OAuth uses standard OAuth 2.0 (no JWT/App auth like GitHub)
- Scopes needed: `read_user` (for profile), `api` (for posting MR comments and reading diffs)
- For self-hosted GitLab: the base URL (`https://gitlab.com`) must be configurable via `GITLAB_BASE_URL` env var
- Reuse the existing session management (HMAC-signed cookies, DB storage) — just add the `platform` field
- State cookie for CSRF protection works identically to the GitHub flow

## GitLab API Differences
- Token exchange returns JSON with `access_token` and `refresh_token` (GitHub doesn't return refresh tokens by default)
- User endpoint: `GET /api/v4/user` returns `{ id, username, avatar_url, ... }`
- No equivalent of GitHub's "installations" — the user's access token directly grants access to their projects

## Acceptance Criteria
- [ ] `/api/auth/gitlab` redirects to GitLab OAuth authorize URL
- [ ] `/api/auth/gitlab/callback` exchanges code and creates a session
- [ ] Session stores `platform: 'gitlab'` and correct user info
- [ ] `/api/auth/me` returns the GitLab user's profile when logged in via GitLab
- [ ] CSRF state validation works correctly
- [ ] Self-hosted GitLab URL is configurable via env var
- [ ] Errors during OAuth flow are handled gracefully (invalid code, network error)
