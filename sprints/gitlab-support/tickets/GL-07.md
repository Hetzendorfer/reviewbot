# GL-07: Refactor Review Pipeline to Use Platform Interfaces

## Description
Refactor `src/review/pipeline.ts` and `src/review/poster.ts` to use the platform abstraction interfaces instead of directly importing from `src/github/`. This makes the pipeline fully platform-agnostic.

## Affected Files
- **Modify**: `src/review/pipeline.ts` — replace `getOctokit`, `createCheckRun`, `fetchPRDiff`, etc. with platform interface calls
- **Modify**: `src/review/poster.ts` — replace `Octokit` param with `PlatformClient`
- **Modify**: `src/repo-config.ts` — replace `Octokit` param with `PlatformClient`
- **Modify**: `src/queue.ts` — job data must include `platform` field to resolve the correct implementation

## Current State (GitHub-coupled)
```typescript
// pipeline.ts
import { getOctokit } from "../github/client.js"
import { createCheckRun, markCheckInProgress, ... } from "../github/checks.js"

const octokit = await getOctokit(job.installationId)
const checkRunId = await createCheckRun(octokit, { ... })
const rawDiff = await fetchPRDiff(octokit, ...)
await postReviewToGitHub(octokit, ...)
```

## Target State (Platform-agnostic)
```typescript
// pipeline.ts
import { getPlatform } from "../platforms/registry.js"
import type { PlatformClient, PlatformStatusReporter } from "../platforms/types.js"

const platform = getPlatform(job.platform)
const client = await platform.getClientForInstallation(job.installationId)
const status = await platform.getStatusReporter(job.installationId)

const statusId = await status.createStatus(owner, repo, headSha, prNumber)
const rawDiff = await client.fetchDiff(owner, repo, job.prNumber)
await postReview(client, owner, repo, ...)
```

## Key Changes

### `pipeline.ts`
1. Import `getPlatform` from registry instead of GitHub-specific modules
2. Resolve `PlatformClient` and `PlatformStatusReporter` based on `job.platform`
3. Replace all `octokit.*` calls with interface method calls
4. Replace `checkRunId: bigint` with `statusId: string`

### `poster.ts`
1. Rename `postReviewToGitHub` → `postReview`
2. Change first param from `Octokit` to `PlatformClient`
3. Call `client.postReview(...)` instead of `postReview(octokit, ...)`

### `repo-config.ts`
1. Change `fetchRepoConfig(octokit, ...)` → `fetchRepoConfig(client: PlatformClient, ...)`
2. Call `client.fetchFileContent(...)` instead of importing from GitHub client

### `queue.ts` / Job Data
1. Add `platform: 'github' | 'gitlab'` to the `JobData` interface
2. Populate from `NormalizedPREvent.platform` when enqueuing
