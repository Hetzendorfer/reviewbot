# GL-01: Define Platform Abstraction Interfaces

## Description
Create a set of TypeScript interfaces that abstract all platform-specific operations (auth, API client, webhook parsing, status reporting). These interfaces are the foundation for multi-platform support â€” all subsequent tickets depend on them.

## Affected Files
- **New**: `src/platforms/types.ts` â€” all platform interfaces and shared types

## Interfaces to Define

### `PlatformClient`
```typescript
interface PlatformClient {
  fetchDiff(owner: string, repo: string, prNumber: number): Promise<string>
  postReview(
    owner: string, repo: string, prNumber: number, commitSha: string,
    body: string, comments: ReviewComment[]
  ): Promise<void>
  fetchFileContent(owner: string, repo: string, path: string, ref: string): Promise<string | null>
}
```

### `PlatformStatusReporter`
```typescript
interface PlatformStatusReporter {
  createStatus(owner: string, repo: string, headSha: string, prNumber: number): Promise<string>
  markInProgress(owner: string, repo: string, statusId: string): Promise<void>
  markSuccess(owner: string, repo: string, statusId: string, summary: string, commentCount: number): Promise<void>
  markFailed(owner: string, repo: string, statusId: string, errorMessage: string): Promise<void>
}
```

### `PlatformWebhookParser`
```typescript
interface PlatformWebhookParser {
  verifySignature(payload: string, signature: string, secret: string): boolean
  parsePREvent(headers: Headers, rawBody: string): NormalizedPREvent | null
}

interface NormalizedPREvent {
  platform: 'github' | 'gitlab'
  installationId: number
  owner: string
  repo: string
  prNumber: number
  prTitle: string
  commitSha: string
  baseBranch: string
  repoFullName: string
  isDraft: boolean
}
```

### `PlatformAuth`
```typescript
interface PlatformAuth {
  getClientForInstallation(installationId: number): Promise<PlatformClient>
  getStatusReporter(installationId: number): Promise<PlatformStatusReporter>
}
```

### `PlatformOAuth`
```typescript
interface PlatformOAuth {
  getAuthorizeUrl(state: string, redirectUri: string): string
  exchangeCode(code: string, redirectUri: string): Promise<{ accessToken: string }>
  getUserInfo(accessToken: string): Promise<{ platformUserId: number; login: string; avatarUrl: string }>
}
```

## Design Decisions
- Use `string` for `statusId` (not `bigint`) since GitLab commit statuses don't return numeric IDs the way GitHub Check Runs do
- `NormalizedPREvent` includes a `platform` discriminator so the queue knows which platform implementation to use
- Keep interfaces minimal â€” only include methods the pipeline actually calls today

## Acceptance Criteria
- [ ] All interfaces are defined and exported from `src/platforms/types.ts`
- [ ] Interfaces cover all operations currently performed via `src/github/` modules
- [ ] No GitHub-specific types leak into the interfaces (no `Octokit`, no GitHub-specific field names)
- [ ] Types compile cleanly with `bunx tsc --noEmit`
