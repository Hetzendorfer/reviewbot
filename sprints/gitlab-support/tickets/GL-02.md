# GL-02: Refactor GitHub Code into Platform Module

## Description
Move and wrap the existing GitHub-specific code (`src/github/`) behind the platform abstraction interfaces defined in GL-01. The existing functionality must remain identical — this is a pure refactor with no behavior changes.

## Affected Files
- **New**: `src/platforms/github/index.ts` — implements all platform interfaces using existing GitHub code
- **Modify**: `src/github/auth.ts` — keep as-is, but wrap in `GitHubPlatformAuth`
- **Modify**: `src/github/client.ts` — keep as-is, but wrap in `GitHubPlatformClient`
- **Modify**: `src/github/checks.ts` — keep as-is, but wrap in `GitHubStatusReporter`
- **Modify**: `src/github/webhooks.ts` — keep as-is, but wrap in `GitHubWebhookParser`

## Implementation Approach

### Option: Wrapper Pattern (Recommended)
Keep the existing `src/github/` files untouched. Create a new `src/platforms/github/index.ts` that imports from `src/github/` and implements the interfaces:

```typescript
import { getOctokit, fetchPRDiff, postReview, fetchFileContent } from "../../github/client.js"
import type { PlatformClient } from "../types.js"

export class GitHubPlatformClient implements PlatformClient {
  constructor(private octokit: Octokit) {}

  async fetchDiff(owner: string, repo: string, prNumber: number): Promise<string> {
    return fetchPRDiff(this.octokit, owner, repo, prNumber)
  }
  // ... etc
}
```

This minimizes risk — the original code paths are preserved and the new interfaces are thin wrappers.

### Registration
Create a `src/platforms/registry.ts` that maps platform names to their implementations:
```typescript
const platforms = { github: GitHubPlatform, gitlab: GitLabPlatform }
export function getPlatform(name: 'github' | 'gitlab'): Platform { ... }
```

## Acceptance Criteria
- [ ] `GitHubPlatformClient` implements `PlatformClient`
- [ ] `GitHubStatusReporter` implements `PlatformStatusReporter`
- [ ] `GitHubWebhookParser` implements `PlatformWebhookParser`
- [ ] `GitHubPlatformAuth` implements `PlatformAuth`
- [ ] All existing GitHub tests still pass
- [ ] No behavioral changes — the GitHub flow works identically
- [ ] Platform registry can resolve `'github'` to the GitHub implementations
