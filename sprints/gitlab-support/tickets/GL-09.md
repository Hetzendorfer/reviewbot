# GL-09: Config — Add GitLab Environment Variables

## Description
Extend the Zod-validated environment config in `src/config.ts` to include optional GitLab configuration variables. All GitLab vars are optional since an operator may run ReviewBot with GitHub-only or GitLab-only.

## Affected Files
- **Modify**: `src/config.ts` — add GitLab env vars to the Zod schema
- **Modify**: `.env.example` — document new variables

## New Environment Variables
| Variable | Type | Required | Description |
|----------|------|----------|-------------|
| `GITLAB_CLIENT_ID` | string | No | GitLab OAuth Application ID |
| `GITLAB_CLIENT_SECRET` | string | No | GitLab OAuth Application Secret |
| `GITLAB_WEBHOOK_SECRET` | string | No | Secret token for verifying GitLab webhook payloads |
| `GITLAB_BASE_URL` | string | No | Base URL for self-hosted GitLab (default: `https://gitlab.com`) |

## Zod Schema Addition
```typescript
// All optional — GitLab support is opt-in
GITLAB_CLIENT_ID: z.string().optional(),
GITLAB_CLIENT_SECRET: z.string().optional(),
GITLAB_WEBHOOK_SECRET: z.string().optional(),
GITLAB_BASE_URL: z.string().url().default("https://gitlab.com"),
```

## Validation Rules
- If `GITLAB_CLIENT_ID` is set, `GITLAB_CLIENT_SECRET` must also be set (and vice versa) — use Zod `.refine()` for this
- `GITLAB_BASE_URL` must be a valid URL if provided
- `GITLAB_WEBHOOK_SECRET` is independent — webhook can work without OAuth (and vice versa)

## `.env.example` Addition
```env
# GitLab (optional — leave blank to disable GitLab support)
GITLAB_CLIENT_ID=
GITLAB_CLIENT_SECRET=
GITLAB_WEBHOOK_SECRET=
GITLAB_BASE_URL=https://gitlab.com
```

## Acceptance Criteria
- [ ] App starts without any GitLab vars (backward compatible)
- [ ] Setting `GITLAB_CLIENT_ID` without `GITLAB_CLIENT_SECRET` fails validation with clear error
- [ ] `GITLAB_BASE_URL` defaults to `https://gitlab.com` when not set
- [ ] `.env.example` documents all new variables
- [ ] `bunx tsc --noEmit` passes
