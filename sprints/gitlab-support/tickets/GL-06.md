# GL-06: Implement GitLab Webhook Handler

## Description
Create a webhook endpoint that receives GitLab MR events and enqueues reviews, mirroring the GitHub webhook handler in `src/api/webhooks/github.ts`.

## Affected Files
- **New**: `src/api/webhooks/gitlab.ts` ŌĆö Elysia plugin for the GitLab webhook route
- **New**: `src/platforms/gitlab/webhooks.ts` ŌĆö GitLab webhook parsing and verification
- **Modify**: `src/index.ts` ŌĆö register the new webhook route

## Endpoint
```
POST /webhooks/gitlab
```

## GitLab Webhook Format
GitLab sends MR events with these characteristics:
- **Header**: `X-Gitlab-Event: Merge Request Hook`
- **Auth**: `X-Gitlab-Token: <secret>` (simple token comparison, not HMAC)
- **Payload** (relevant fields):
```json
{
  "object_kind": "merge_request",
  "event_type": "merge_request",
  "project": {
    "id": 123,
    "path_with_namespace": "acme/widget",
    "web_url": "https://gitlab.com/acme/widget"
  },
  "object_attributes": {
    "iid": 45,
    "title": "Add caching",
    "state": "opened",
    "action": "open",
    "draft": false,
    "last_commit": { "id": "abc123..." },
    "target_branch": "main",
    "source_branch": "feature/cache"
  }
}
```

## Relevant MR Actions
| GitLab Action | GitHub Equivalent | Should Trigger Review? |
|--------------|------------------|----------------------|
| `open` | `opened` | Yes |
| `update` (with commit changes) | `synchronize` | Yes |
| `reopen` | `reopened` | Yes |
| `merge` | ŌĆö | No |
| `close` | ŌĆö | No |

Note: GitLab's `update` action fires for many changes (title, labels, assignees, etc.). Only trigger a review when `oldrev` is present in the payload (indicating new commits were pushed).

## Signature Verification
GitLab uses a simple secret token comparison (not HMAC):
```typescript
function verifyGitLabWebhook(headers: Headers, secret: string): boolean {
  return headers.get("x-gitlab-token") === secret
}
```
This is simpler than GitHub's HMAC but less secure. Use timing-safe comparison to prevent timing attacks.

## Normalization to `NormalizedPREvent`
```typescript
{
  platform: 'gitlab',
  installationId: payload.project.id,  // GitLab project ID serves as installation ID
  owner: payload.project.path_with_namespace.split('/')[0],
  repo: payload.project.path_with_namespace.split('/').slice(1).join('/'),
  prNumber: payload.object_attributes.iid,
  prTitle: payload.object_attributes.title,
  commitSha: payload.object_attributes.last_commit.id,
  baseBranch: payload.object_attributes.target_branch,
  repoFullName: payload.project.path_with_namespace,
  isDraft: payload.object_attributes.draft
}
```

## Acceptance Criteria
- [ ] `POST /webhooks/gitlab` accepts GitLab MR webhook payloads
- [ ] Secret token is verified via timing-safe comparison
- [ ] Only `open`, `update` (with commits), and `reopen` actions trigger reviews
- [ ] Draft MRs are skipped
- [ ] Payload is normalized to `NormalizedPREvent` and enqueued
- [ ] Non-MR events are ignored (returns 200 with `{ ignored: true }`)
- [ ] Invalid or missing token returns 401
